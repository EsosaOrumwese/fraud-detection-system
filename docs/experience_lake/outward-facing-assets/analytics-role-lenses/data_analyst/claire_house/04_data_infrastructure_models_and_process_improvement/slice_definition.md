This maps directly to the Claire House candidate-profile requirement that the analyst should be able to describe:
- supporting efficient data processing and retrieval for analytics
- maintaining analytical data structures rather than only consuming them
- noticing where data-capture controls weaken downstream analytical quality

`good with infrastructure`

- maintainable data-model support
- capture-control awareness
- efficiency and automation improvement under the reporting layer

### 8.2 Lens 03 - Surface one capture or control weakness

This is the main support lens for trust and control.

Tasks:
1. Identify one control point where weak capture or processing logic would damage downstream reuse.
2. Profile the weakness in a bounded way:
- missingness
- duplicate risk

- manual step risk
- uncontrolled field selection
3. Record one control-monitoring note:
- where the weakness sits
- why it matters
- how the maintained layer reduces or contains it

### 8.3 Lens 09 - Make the maintained layer repeatable

This is what turns the slice into believable analytical-layer support rather than one-off tidying.

Tasks:
1. Stabilise the maintained layer definition:
- inputs
- outputs
- shape
- allowed exclusions
- rerun rule
2. Version the shaping logic:
- query or transformation logic
- check logic
- release logic
3. Create one rerun checklist:
- what has to exist before rebuild
- what checks must pass
- what would break comparability
4. Document caveats:
- what the maintained layer is suitable for
- what it should not be used for
- what changes would invalidate reuse

### 8.4 Lens 05 - State the efficiency improvement clearly

This is light support only.

Tasks:
1. Write one short efficiency-improvement note:
- what manual or repeated step has been reduced

2. Keep it bounded:
- no broad transformation programme claim

- one concise improvement statement only

## 1. Requirement Being Targeted

The requirement being targeted here is the Welsh Government `Payroll Data Analyst` expectation around:
- validating detailed operational or payroll-style data
- identifying discrepancies and resolving them promptly
- preparing and supporting reconciliation reporting
- supporting recurring operational processing cycles with accurate and defensible outputs

This maps directly to the candidate-profile requirement that the analyst should be able to sound like someone who:
- can work confidently with detailed records in a high-consequence environment
- can validate data against rules or expected outcomes
- can identify mismatches before they damage payment or reporting integrity
- can support reconciliation and recurring reporting cycles reliably

This note exists because that requirement is easy to flatten into the vague statement:
`good at operational reporting`

That is not the real burden.

The employer is asking for something sharper:
- validation of detailed or controlled records
- discrepancy detection and handling
- reconciliation output support
- repeatable operational-control delivery inside a cyclical processing environment

## 2. The Analogue On The Current Platform

The current platform does not contain a literal payroll-processing estate, statutory pay engine, or live `HMRC` compliance environment, so the slice has to use an honest analogue.

The closest truthful analogue is:
- one recurring operational dataset or controlled operational lane
- one explicit validation surface over that lane
- one bounded discrepancy class or mismatch condition
- one reconciliation-ready summary or balancing surface
- one repeatable release-safe output path suitable for recurring operational use

So for this slice:
- `analyse payroll data to ensure accuracy` maps to `validate one recurring operational lane for expected internal consistency`
- `identify discrepancies and resolve issues` maps to `surface one mismatch class clearly enough to support correction or controlled follow-up`
- `prepare reconciliation reports` maps to `produce one reconciliation-style balancing view`
- `support monthly payroll cycles` maps to `package the result in one repeatable recurring-support pack`

The claim must therefore remain:
- control shaped
- validation shaped
- reconciliation shaped
- platform-truth bounded
- careful not to imply literal payroll ownership, statutory authority, or end-to-end pay processing where this slice only proves one bounded operational-control pack

Default first-pass posture:
- aggregate-control and balancing first
- discrepancy class first
- stronger row-level payroll-process language only if the bounded evidence genuinely requires it

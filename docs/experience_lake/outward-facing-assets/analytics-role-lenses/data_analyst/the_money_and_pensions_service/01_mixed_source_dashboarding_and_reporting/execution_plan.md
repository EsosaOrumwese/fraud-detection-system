## 2. Mixed-Source-First Posture

- identify the common reporting grain second
- define the shared dimensions and combined logic third
- build the dashboard-style summary fourth
- build the supporting report-style detail output fifth
- write the audience-use and caveat notes sixth

This matters because the Money and Pensions Service responsibility is about reporting built from several evidence streams, and the execution should read like cross-source synthesis rather than single-lane dashboard production.

## 2A. Memory And Query Discipline

This slice must be executed with explicit memory discipline.

The correct posture is:
- do not pull broad raw surfaces into pandas
- do not reopen wide raw profiling if the mixed-source reporting question can be answered from compact governed outputs
- do not assume “multiple systems” justifies a broad rebuild when the proof can be made from controlled compact surfaces

The correct query discipline is:
- start with compact-output profiling before any new materialisation
- use inherited summaries, supporting cuts, and audience-shaped outputs first
- if a probe is needed, filter the window and fields at scan time
- materialise compact integrated outputs only after the common reporting grain and shared dimensions are clear
- let Python read only those compact outputs, not raw detailed source families

The default execution ceiling for this slice is:
- inherited governed surfaces first
- one compact integrated reporting layer second
- one compact dashboard summary and one compact detail output third
- one compact note set fourth

If a query starts behaving like a full customer-experience reporting rebuild rather than a bounded mixed-source proof, stop and narrow it before proceeding.

## 4. Early Scope Gate

Before building either the dashboard-style summary or the report-style detail output, run a bounded scope gate.

The first profiling pass must answer:
- which governed evidence surfaces are strong enough to combine honestly?
- what common reporting grain can represent them without distorting their meaning?
- which shared dimensions are necessary to align them?
- can the dashboard summary and supporting detail output both be produced from the same compact combined logic?
- do the row counts, fields, and current notes show that the slice can remain compact?

Required checks:
- continuity of the chosen compact evidence surfaces
- continuity of shared dimensions across those surfaces
- ability to define one common reporting grain
- ability to produce a summary view and a supporting detail cut without diverging logic

Those checks should be implemented as:
- aggregate-only or compact-output probes
- inherited compact outputs where available
- no broad row materialisation into memory

Decision rule:
- if the inherited surfaces are explicit enough, proceed with the bounded mixed-source pack
- if not, add one compact integration layer rather than broadening the query footprint

## 5. Candidate Analytical Base

The first-pass analytical base should stay bounded and explicit.

Each component should have one stated purpose:
- compact evidence surface one
  - provide the first reporting angle
- compact evidence surface two
  - provide the second reporting angle

## 8. Common-Grain Strategy

The combined reporting grain has to stay honest.

The first-pass grain strategy should answer:
- what is the shared reporting grain?
- why is it acceptable for all chosen evidence surfaces?
- which dimensions are preserved across the combined logic?
- why does that grain make the integrated pack more useful than source-by-source outputs?

Allowed grain posture:
- one compact shared analytical grain that preserves the meaning of each retained surface
- one integrated summary and one supporting detail cut from that same grain

Disallowed grain posture:
- forcing incompatible surfaces together simply to say the pack is mixed-source

Decision rule:
- every retained field must earn its place in either:
  - integrated summary
  - supporting detail
  - audience-use explanation
- if a field does not support one of those purposes, drop it
